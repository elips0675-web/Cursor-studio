# Тестирование и производительность в SwiftMatch

В проекте реализован комплексный подход к обеспечению качества и скорости работы приложения.

---

### 1. Автоматизированное тестирование

*   **Unit-тесты (Jest):** Проверка бизнес-логики (алгоритм автопоиска, фильтры мата, утилиты обработки текста).
    *   `npm run test`
*   **E2E-тесты (Playwright):** Имитация действий пользователя: Регистрация -> Свайп -> Мэтч -> Чат.
    *   `npx playwright test`
*   **Нагрузочное тестирование (K6):** Проверка стабильности Firestore при 10,000+ одновременных соединениях.

### 2. Оптимизация производительности

Для достижения высоких показателей в Google PageSpeed Insights (LCP < 1.5s) применены следующие техники:

*   **Приоритизация LCP:** Основные изображения анкет имеют атрибут `priority`.
*   **Bundle Optimization:** Использование `optimizePackageImports` для сокращения размера JS-бандла.
*   **Dynamic Rendering:** Тяжелые интерактивные элементы (графики Recharts, сложные модальные окна) подгружаются только на клиенте через `next/dynamic`.
*   **Prefetching:** Next.js фоново подгружает код для навигационных ссылок, что делает переходы мгновенными.

### 3. Мониторинг в реальном времени

В панели администратора (`/admin/analytics`) доступен блок **«Техническое здоровье»**, который отображает:
*   **Uptime:** Стабильность работы сервисов.
*   **LCP Speed:** Реальное время отрисовки контента у пользователей.
*   **API Latency:** Задержка ответов от Firebase и AI-моделей.
